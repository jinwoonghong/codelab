# Meeting Scheduler 프로젝트 기획서

## 프로젝트 개요

### 프로젝트명
**Meeting Scheduler** (미팅 스케줄러)

### 한 줄 설명
카카오톡 단체채팅방에서 링크 하나로 회의 일정을 투표로 정하는 초간단 모바일웹

### 배경 및 문제점
- 단체채팅방에서 "언제 시간 되세요?" 물으면 **답변이 제각각**으로 와서 정리하기 어려움
- "저는 월요일, 화요일 가능해요~" → 결국 **누군가가 엑셀로 정리**해야 함
- 기존 서비스(Doodle 등)는 **PC 중심 UI**, 모바일에서 불편
- 카카오톡에서 링크 클릭 → **바로 투표 가능**해야 하는데, 기존 서비스는 절차가 복잡
- **핵심 문제**: "5명 이상 일정 맞추기가 너무 귀찮다"

### 해결 방안
**"링크 공유 → 투표 → 확정"**

1. 회의 주최자가 투표 생성 (30초 완료)
2. 카카오톡에 링크 공유
3. 참여자들이 링크 클릭 → 가능한 시간 투표 (10초 완료)
4. 실시간으로 결과 확인
5. 가장 많이 겹치는 시간으로 확정

→ **앱 설치 없이, 링크만으로 일정 조율 완료**

### 목표
- **초간단 생성**: 30초 안에 투표 링크 생성
- **즉시 참여**: 앱 설치/회원가입 없이 링크만으로 투표
- **실시간 확인**: 투표 현황 실시간 업데이트
- **모바일 최적화**: 카카오톡 인앱 브라우저에서 완벽 동작
- **예쁜 공유**: OG 태그로 카카오톡 미리보기 카드 지원

---

## 타겟 사용자

### 주 타겟
- 회사 팀 회의/회식 일정 잡는 직장인
- 스터디/동아리 모임 시간 정하는 대학생
- 동창회/모임 날짜 정하는 모든 사람
- 5명 이상 그룹에서 일정 조율이 필요한 누구나

### 핵심 사용 시나리오

#### 시나리오 1: 팀 점심 회식 일정 잡기
```
팀장: "다음 주 회식 언제 가능하신가요?"
      (매번 이 질문에 답변 정리하기 귀찮음)
         ↓
Meeting Scheduler에서 투표 생성
- 제목: "팀 회식 일정"
- 날짜: 12/9(월), 12/10(화), 12/11(수)
- 시간: 저녁 6시, 저녁 7시
- 장소: 강남역, 역삼역 (선택)
         ↓
생성된 링크를 카카오톡 단톡방에 공유
"회식 일정 투표 부탁드려요~ [링크]"
         ↓
팀원들이 링크 클릭 → 이름 입력 → 가능한 시간 체크
         ↓
실시간으로 표 업데이트
"12/10(화) 저녁 7시 - 5명 가능 ✓"
         ↓
팀장이 최다 참여 시간으로 확정
"12/10 저녁 7시 역삼역으로 확정합니다!"
```

#### 시나리오 2: 대학 스터디 시간 정하기
```
스터디장: "이번 주 스터디 언제 할까요?"
         ↓
투표 생성
- 제목: "알고리즘 스터디"
- 날짜/시간: 토 오후 2시, 토 오후 5시, 일 오전 10시
- 장소: 스터디카페 A, 학교 도서관
         ↓
카톡에 링크 공유
         ↓
5명 중 4명이 "토 오후 5시, 스터디카페 A" 선택
         ↓
자동으로 최다 득표 확인 → 확정
```

#### 시나리오 3: 고등학교 동창 모임
```
총무: "연말 동창회 언제 가능하신 분?"
      (30명 카톡방에서 일일이 확인 불가능)
         ↓
투표 생성
- 제목: "○○고 20기 연말 동창회"
- 날짜: 12/21(토), 12/28(토), 1/4(토)
- 시간: 저녁 6시
- 장소: 강남, 홍대, 신촌
         ↓
링크 공유 + "투표해주세요~"
         ↓
30명 중 20명 투표 완료
         ↓
12/28(토) 강남이 최다 → 확정 공지
```

## 핵심 가치 제안

### "링크 하나로 일정 조율 끝"

1. **30초 생성**: 제목 + 옵션만 입력하면 끝
2. **10초 투표**: 이름 입력 → 체크 → 완료
3. **설치 불필요**: 웹 링크만으로 동작
4. **회원가입 불필요**: 이름만 입력하면 투표 가능
5. **실시간 결과**: 누가 언제 가능한지 한눈에
6. **카톡 최적화**: 미리보기 카드로 예쁘게 공유

## 주요 기능 목록

### Phase 1: MVP (필수 기능)

#### 1. 투표 생성 페이지
- **투표 제목** 입력 (필수)
  - 예: "팀 회식 일정", "스터디 모임"
- **날짜/시간 옵션** 추가 (필수, 최소 2개)
  - 캘린더 UI로 날짜 선택
  - 시간 선택 (30분 단위)
  - 여러 옵션 추가 가능
- **장소 옵션** 추가 (선택)
  - 텍스트 입력으로 장소명 입력
  - 여러 장소 추가 가능
- **주최자 이름** 입력
- **생성하기** 버튼 → 투표 링크 생성

#### 2. 투표 링크 생성 및 공유
- **고유 투표 ID** 생성 (예: `meet.ing/abc123`)
- **공유용 URL** 생성
- **복사하기** 버튼 (클립보드 복사)
- **카카오톡 공유** 버튼 (Kakao SDK 연동)
- **OG 태그** 설정
  - 제목: "투표해주세요: [투표 제목]"
  - 설명: "현재 N명 참여 중"
  - 이미지: 서비스 로고 또는 캘린더 아이콘

#### 3. 투표 참여 페이지
- **투표 제목** 표시
- **주최자** 표시 ("홍길동님이 만든 투표")
- **참여자 이름** 입력 (필수)
- **옵션 목록** 표시 (날짜/시간 + 장소)
  - 체크박스 형태로 복수 선택 가능
  - "가능" / "불가능" / "아마도" 3단계 (선택)
- **투표하기** 버튼
- **이미 투표한 경우**: 수정 가능

#### 4. 투표 결과 페이지
- **투표 제목** 및 **주최자**
- **전체 참여자 수** 표시
- **옵션별 투표 현황**
  - 날짜/시간별 가능한 사람 수
  - 프로그레스 바로 시각화
  - 최다 득표 옵션 하이라이트
- **참여자 매트릭스** 표시
  ```
           12/9(월)  12/10(화)  12/11(수)
  홍길동      ✓         ✓
  김철수      ✓         ✓         ✓
  이영희                ✓         ✓
  ```
- **실시간 업데이트** (새로운 투표 시 자동 갱신)

#### 5. 투표 마감 및 확정
- **주최자 전용**: 투표 마감 기능
- **확정하기** 버튼 → 최종 일정 선택
- **확정 공지** 생성
  - "12/10(화) 저녁 7시 역삼역으로 확정되었습니다!"
- 확정 후에는 투표 참여 불가

#### 6. 데이터 저장
- **Supabase 또는 Firebase** 사용
  - 투표 정보 (제목, 옵션, 주최자)
  - 참여자 투표 내역
  - 투표 상태 (진행중/마감/확정)
- **MVP 대안**: LocalStorage + URL 파라미터
  - 서버 없이 클라이언트만으로 동작
  - 단, 실시간 동기화 불가

### Phase 2: 고급 기능

1. **카카오톡 공유 API**
   - 카카오 SDK 연동
   - 커스텀 템플릿 메시지
   - 공유 시 예쁜 카드 생성

2. **투표 알림**
   - 투표 마감 임박 알림
   - 새로운 참여자 알림 (주최자에게)
   - 확정 알림 (참여자들에게)

3. **댓글/메모 기능**
   - 옵션별 메모 가능
   - "이 날은 오후 5시 이후만 가능해요"

4. **익명 투표**
   - 이름 대신 익명 옵션
   - 결과에서 "익명 3명" 표시

5. **반복 투표 템플릿**
   - 정기 회의용 템플릿 저장
   - "매주 스터디 일정" 빠르게 생성

---

### Phase 3: 프리미엄 기능

1. **구글 캘린더 연동**
   - 확정된 일정 자동 등록
   - 참여자 캘린더에 초대

2. **슬랙/디스코드 연동**
   - 봇으로 투표 생성/결과 공유
   - 채널 내에서 투표 가능

3. **회의록 연동**
   - 확정된 회의에 회의록 작성
   - 참석자에게 공유

4. **팀 워크스페이스**
   - 팀 단위 투표 관리
   - 과거 투표 히스토리

## 디자인 방향

### UI/UX 원칙
1. **3스텝 완료**: 생성/투표 모두 3단계 이내
2. **엄지 영역**: 중요 버튼은 하단에 배치 (한 손 조작)
3. **큰 터치 영역**: 체크박스, 버튼 모두 44px 이상
4. **명확한 피드백**: 투표 완료 시 애니메이션/햅틱
5. **카톡 친화적**: 인앱 브라우저에서 완벽 동작

### 컬러 스킴
- **Primary**: 카카오 옐로우 계열 (#FEE500) - 친근함
- **Secondary**: 블루 계열 (#3182F6) - 신뢰감
- **Success**: 그린 계열 (#00C471) - 확정/가능
- **Warning**: 오렌지 계열 (#FF9500) - 마감 임박
- **Neutral**: 그레이 계열 - 비활성화

### 폰트
- **모바일 최적화**: Pretendard 또는 시스템 폰트
- **가독성**: 본문 16px 이상

## 기술 스택

### Frontend
- **HTML5 / CSS3 / JavaScript (ES6+)**
- **Vite** - 빌드 도구
- **Vanilla JS** 또는 **Preact** (가벼운 프레임워크)

### Backend (선택)
- **Supabase** - BaaS (추천)
  - PostgreSQL 기반
  - 실시간 구독 기능 내장
  - 인증 기능 (선택적)
- 또는 **Firebase** - BaaS
  - Firestore 실시간 DB
  - 무료 티어 충분

### MVP 대안 (서버리스)
- **LocalStorage + URL 해시**
  - 투표 데이터를 URL에 인코딩
  - 실시간 동기화 불가하지만 MVP로는 충분
  - 예: `meet.ing/#data=eyJ0aXRsZSI6Ii4uLiJ9`

### 배포
- **Vercel** 또는 **Netlify** (무료)
- 커스텀 도메인 연결

### 외부 API
- **Kakao SDK** - 카카오톡 공유 (Phase 2)
- **Google Calendar API** - 캘린더 연동 (Phase 3)

```javascript
{
  id: "abc123",              // 고유 ID

  // 날짜/시간 옵션
  timeOptions: [
    { id: "t1", datetime: "2025-12-09T19:00:00" },
    { id: "t2", datetime: "2025-12-10T19:00:00" },
    { id: "t3", datetime: "2025-12-11T19:00:00" }
```
