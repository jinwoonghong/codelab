# Link Keeper 프로젝트 기획서

## 📌 프로젝트 개요

### 프로젝트명
**Link Keeper** (링크 키퍼)

### 한 줄 설명
모바일에서 수집한 링크를 스마트하게 관리하고 나중에 편하게 찾아볼 수 있는 모바일 웹 앱

### 배경 및 문제점
- 모바일로 유튜브, 쓰레드(Thread), SNS, 블로그 등을 보다가 나중에 보려고 링크를 저장
- 저장한 링크들이 시간이 지나면서 쌓이고 관리가 어려워짐
- 어떤 링크를 확인했는지 안 했는지 구분하기 어려움
- 날짜별로 정리가 안 되어 있어서 찾기 힘듦
- 기존 북마크 기능은 폴더 구조로 되어 있어 모바일에서 사용하기 불편

### 목표
- 모바일 환경에 최적화된 링크 관리 시스템 구축
- 직관적인 UI/UX로 쉽고 빠르게 링크를 저장하고 관리
- 읽음/안 읽음 상태를 명확하게 구분
- 날짜별, 카테고리별 정리 기능 제공
- 오프라인에서도 동작하는 PWA 앱

---

## 👥 타겟 사용자

### 주 타겟
- 모바일로 다양한 콘텐츠를 소비하는 20-40대
- 나중에 읽기 위해 링크를 자주 저장하는 사람
- 정보를 체계적으로 관리하고 싶은 사람

### 사용 시나리오
1. **출퇴근 중**: 지하철에서 재미있는 유튜브 영상을 발견 → 빠르게 저장
2. **업무 중**: 참고할 만한 블로그 글 발견 → 카테고리와 함께 저장
3. **저녁 시간**: 저장한 링크 목록을 보면서 하나씩 확인하고 읽음 처리
4. **주말**: 이번 주에 저장한 링크들을 정리하고 불필요한 것 삭제

## 🎯 핵심 가치 제안

1. **빠른 저장**: 3초 이내로 링크 저장 가능
2. **명확한 상태**: 읽음/안 읽음이 시각적으로 명확
3. **스마트한 정리**: 자동으로 날짜별 그룹화
4. **모바일 최적화**: 터치 친화적인 UI
5. **오프라인 지원**: 네트워크 없이도 사용 가능

## 📋 주요 기능 목록

### Phase 1: MVP (필수 기능)
1. **링크 추가**
   - URL 입력 및 저장
   - 제목 자동 추출 (메타 데이터)
   - 간단한 메모 추가

2. **링크 목록 보기**
   - 날짜별 그룹화 (오늘, 어제, 이번 주, 이번 달, 이전)
   - 읽음/안 읽음 상태 표시
   - 썸네일 이미지 표시

3. **링크 상태 관리**
   - 읽음 처리
   - 안 읽음으로 되돌리기
   - 삭제

4. **필터링**
   - 전체 보기
   - 안 읽은 것만 보기
   - 읽은 것만 보기

### Phase 2: 고급 기능
1. **카테고리/태그**
   - 커스텀 카테고리 생성
   - 여러 태그 추가
   - 카테고리별 필터링

2. **검색**
   - 제목, URL, 메모로 검색
   - 최근 검색어 저장

3. **정렬**
   - 날짜순 (최신/오래된 순)
   - 제목순
   - 상태별

4. **편집**
   - 제목 수정
   - 메모 수정
   - 카테고리 변경

### Phase 3: 고급 UX
1. **대량 작업**
   - 여러 링크 선택
   - 일괄 읽음 처리
   - 일괄 삭제
   - 일괄 카테고리 지정

2. **통계**
   - 총 저장된 링크 수
   - 읽은/안 읽은 링크 수
   - 카테고리별 통계
   - 이번 주/월 저장 추이

3. **공유 기능**
   - 다른 앱에서 링크 공유 받기 (Web Share Target API)
   - 링크 내보내기 (JSON, CSV)

### Phase 4: 프리미엄 기능 (선택)
1. **동기화**
   - 클라우드 백업
   - 여러 기기 간 동기화
   - 로그인/회원가입

2. **AI 기능**
   - 링크 내용 요약
   - 자동 카테고리 제안
   - 유사 링크 추천

3. **알림**
   - 안 읽은 링크 알림
   - 주간 리마인더

## 🎨 디자인 방향

### UI/UX 원칙
1. **미니멀리즘**: 필요한 정보만 간결하게
2. **터치 친화적**: 최소 44x44px 터치 영역
3. **명확한 시각적 피드백**: 버튼 클릭, 상태 변경 시 즉각 반응
4. **빠른 액션**: 스와이프 제스처로 빠른 작업 수행

### 컬러 스킴
- **Primary**: 블루 계열 (#2196F3) - 신뢰감, 집중
- **Success**: 그린 계열 (#4CAF50) - 읽음 처리
- **Neutral**: 그레이 계열 - 배경, 구분선
- **Warning**: 오렌지 계열 - 안 읽음 강조

### 레이아웃
- **카드형 레이아웃**: 각 링크를 카드로 표현
- **그룹 헤더**: 날짜별 구분이 명확한 헤더
- **하단 네비게이션**: 주요 메뉴는 하단에 배치 (모바일 엄지 손가락 영역)

## 📊 성공 지표 (KPI)

### 사용성 지표
- 링크 저장 완료까지 평균 시간: 10초 이하
- 일일 활성 사용자(DAU) 리텐션: 40% 이상
- 주간 활성 사용자(WAU): 첫 달 50명 이상

### 기능 사용 지표
- 저장된 평균 링크 수: 사용자당 20개 이상
- 읽음 처리율: 저장된 링크의 60% 이상
- 검색 기능 사용률: 주간 활성 사용자의 30% 이상

### 기술 지표
- 앱 로딩 시간: 2초 이내
- 오프라인 동작: 100%
- 모바일 호환성: iOS Safari, Android Chrome 완벽 지원

## 💡 차별화 포인트

### 기존 솔루션 대비 장점

| 솔루션 | 단점 | Link Keeper의 장점 |
|--------|------|-------------------|
| 브라우저 북마크 | 모바일에서 사용 불편, 읽음 상태 없음 | 모바일 최적화, 상태 관리 |
| Pocket | 무겁고 복잡함, 유료 | 가볍고 간단함, 무료 |
| 메모 앱 | 링크 관리에 특화되지 않음 | 링크 전용 기능 |
| 카카오톡 나와의 채팅 | 검색/정리 어려움 | 체계적인 관리 |

### 우리만의 강점
1. **초간단 UX**: 3초 안에 저장
2. **한눈에 보이는 읽음 상태**: 시각적으로 명확
3. **자동 날짜 그룹화**: 별도 정리 불필요
4. **모바일 제스처 지원**: 스와이프로 빠른 작업
5. **완전 오프라인**: 네트워크 불필요

## ⚠️ 리스크 및 대응 방안

### 기술적 리스크
- **리스크**: 브라우저 호환성 문제
- **대응**: 주요 브라우저 테스트, polyfill 사용

- **리스크**: LocalStorage 용량 제한 (약 5-10MB)
- **대응**: IndexedDB 사용, 데이터 압축

### 비즈니스 리스크
