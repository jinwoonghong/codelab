# Link Keeper 프로젝트 기획서

## 📌 프로젝트 개요

### 프로젝트명
**Link Keeper** (링크 키퍼)

### 한 줄 설명
모바일 웹 탐색 중 발견한 링크를 공유 버튼 한 번으로 저장하고 체계적으로 관리하는 PWA

### 배경 및 문제점
- 모바일로 유튜브, 쓰레드(Thread), SNS, 블로그 등을 보다가 **나중에 보려고 링크를 저장하고 싶은데 방법이 마땅치 않음**
- 카카오톡 나와의 채팅에 저장? → 찾기 어렵고 관리 안 됨
- 브라우저 북마크? → 모바일에서 폴더 탐색 불편하고, 읽음/안 읽음 구분 없음
- 메모앱에 복붙? → 링크 관리에 특화되지 않아 불편
- **핵심 문제**: 탐색 중인 앱에서 벗어나지 않고 빠르게 저장할 방법이 없음

### 해결 방안
**"공유 버튼만 누르면 끝!"**

1. 어떤 앱/웹에서든 공유 버튼(⋮, 공유 아이콘) 클릭
2. 시스템 공유 메뉴에서 "Link Keeper" 선택
3. 자동으로 링크 저장 완료 (토스트 알림)
4. 나중에 Link Keeper 앱 열어서 편하게 확인

→ **앱 전환 없이, 2탭만으로 저장 완료**

### 목표
- **초간단 저장**: 공유 버튼 → Link Keeper 선택 (2초 완성)
- **시스템 수준 통합**: iOS/Android 공유 메뉴에 네이티브처럼 통합
- **스마트한 관리**: 자동 정리, 읽음/안 읽음 구분
- **완전 오프라인**: PWA로 네트워크 없이도 동작
- **모바일 최적화**: 터치 친화적 UI/UX

## 👥 타겟 사용자

### 주 타겟
- 모바일로 다양한 콘텐츠를 소비하는 20-40대
- "나중에 봐야지" 하고 링크를 자주 저장하는 사람
- 정보를 체계적으로 관리하고 싶지만 복잡한 건 싫어하는 사람

### 핵심 사용 시나리오 (공유 중심)

#### 시나리오 1: 출퇴근 중 유튜브 영상 저장
```
지하철에서 유튜브 탐색 중
 ↓
재미있는 영상 발견! (하지만 데이터 아껴야 함)
 ↓
[공유 버튼] → [Link Keeper] 선택
 ↓
"링크가 저장되었습니다" (토스트)
 ↓
집에 와서 WiFi로 Link Keeper 열어서 시청
```

#### 시나리오 2: 트위터에서 블로그 글 저장
```
트위터 타임라인 스크롤 중
 ↓
흥미로운 블로그 링크 발견
 ↓
[공유 버튼] → [Link Keeper]
 ↓
저장 완료, 계속 타임라인 탐색
 ↓
저녁에 Link Keeper에서 차근차근 읽기
```

#### 시나리오 3: 크롬 브라우저에서 뉴스 저장
```
구글 검색 중 유용한 기사 발견
 ↓
크롬 주소창 옆 [공유] 아이콘
 ↓
[Link Keeper] 선택 + 간단 메모 추가 (선택)
 ↓
저장 완료
 ↓
주말에 저장한 기사들 몰아보기
```

#### 시나리오 4: 주말에 저장한 링크 정리
```
Link Keeper 앱 실행
 ↓
이번 주 저장한 링크 15개 확인
 ↓
하나씩 읽으면서 ✓ 읽음 처리
 ↓
쓸데없는 거 삭제
 ↓
중요한 건 카테고리 지정
```

## 🎯 핵심 가치 제안

### "공유만 하면 관리는 자동으로"

1. **2초 저장**: 공유 버튼 2번 탭으로 저장 완료
2. **앱 전환 없음**: 현재 보던 앱 그대로, 흐름이 끊기지 않음
3. **시스템 통합**: 카카오톡, 메시지와 나란히 있는 네이티브 경험
4. **자동 정리**: 날짜별 그룹화, 메타데이터 자동 추출
5. **명확한 상태**: 읽음/안 읽음 한눈에 구분
6. **완전 오프라인**: 설치 후 인터넷 없어도 작동

   - 홈 화면에 추가 안내

**이것이 핵심! 공유로 받는 게 메인 입력 방식**

#### 2. 공유받기 및 링크 저장
   - 다른 앱에서 공유한 링크 수신
   - 자동 메타데이터 추출
     - 페이지 제목 (og:title, title 태그)

     - 도메인 이름

   - IndexedDB에 저장
   - 저장 완료 토스트 알림

#### 3. 링크 목록 보기
   - 날짜별 자동 그룹화
     - 오늘
     - 어제
     - 이번 주
     - 이번 달
     - 이전
   - 링크 카드 디자인
     - 썸네일 이미지
     - 제목
     - 도메인 이름 (youtube.com, blog.com 등)
     - 저장 시간 (상대 시간: 5분 전, 2시간 전)
     - 읽음/안 읽음 상태 아이콘
   - 빈 상태 UI ("공유 버튼으로 링크를 추가해보세요!")

#### 4. 링크 상태 관리
   - 읽음 처리 (체크박스 또는 토글)
   - 안 읽음으로 되돌리기
   - 시각적 구분
     - 읽은 링크: 흐린 색상, ✓ 아이콘
     - 안 읽은 링크: 선명한 색상, ○ 아이콘
   - 읽은 시간 기록

#### 5. 필터링
   - 전체 보기
   - 안 읽은 것만 보기 (기본값 추천)
   - 읽은 것만 보기
   - 탭 형태로 빠른 전환

#### 6. 링크 삭제
   - 개별 링크 삭제
   - 삭제 확인 다이얼로그
   - 스와이프 제스처로 삭제 (선택)

#### 7. 링크 열기
   - 카드 탭 → 링크 상세 화면
   - "링크 열기" 버튼 → 새 탭에서 원본 페이지 열림
   - 링크 복사 버튼

#### 8. 앱 내에서도 직접 추가 가능 (보조 기능)
   - + 버튼으로 수동 추가 모달
   - URL 입력 및 저장
   - (주된 방식은 공유지만, 옵션으로 제공)

```json
{
  "name": "Link Keeper",
  "share_target": {
    "action": "/share",
    "method": "POST",
    "enctype": "application/x-www-form-urlencoded",
    "params": {
      "title": "title",
      "text": "text",
      "url": "url"
    }
  }
}
```

## 📊 성공 지표 (KPI)

### 사용성 지표
- **공유로 저장 완료까지 평균 시간**: 5초 이하
- **공유 기능 사용 비율**: 전체 저장의 80% 이상 (이게 메인!)
- **일일 활성 사용자(DAU) 리텐션**: 40% 이상
- **주간 활성 사용자(WAU)**: 첫 달 50명 이상

### 기능 사용 지표
- **사용자당 평균 저장 링크 수**: 20개 이상
- **읽음 처리율**: 저장된 링크의 60% 이상
- **PWA 설치율**: 방문자의 30% 이상

### 기술 지표

## 💡 차별화 포인트

### 기존 솔루션 대비 장점

| 솔루션 | 단점 | Link Keeper의 장점 |
|--------|------|-------------------|
| 브라우저 북마크 | 공유로 못 받음, 앱 전환 필요 | 시스템 공유 메뉴에 통합 |
| Pocket | 무겁고 복잡, 계정 필요 | 가볍고 간단, 로컬 저장 |
| 카카오톡 나와의 채팅 | 검색/관리 어려움 | 링크 전용 관리 기능 |
| 메모 앱 | 링크에 특화 안 됨 | 메타데이터 자동 추출 |

### 우리만의 강점
1. **시스템 수준 통합**: 공유 메뉴에 네이티브처럼 등장
2. **2초 저장**: 공유 → 선택 → 완료
3. **앱 전환 없음**: 현재 흐름이 끊기지 않음
4. **완전 무료 오프라인**: 설치만 하면 평생 공짜
5. **초간단 UX**: 배울 게 없음

## ⚠️ 기술적 제약 및 대응

### Web Share Target API 제약사항

#### 제약 1: HTTPS 필수
- **대응**: Vercel/Netlify로 배포 (자동 HTTPS)

#### 제약 2: PWA 설치 필수
- **대응**:
  - 명확한 설치 가이드 제공
  - "홈 화면에 추가해주세요" 안내
  - 설치 전 기능 제한 (공유 받기 불가 안내)

#### 제약 3: iOS Safari 제한적 지원
- **대응**:
  - iOS 13+ 에서 지원 확인
  - 미지원 브라우저에는 수동 추가 안내
  - Progressive Enhancement 전략

#### 제약 4: 일부 앱에서 공유 안 될 수 있음
- **대응**:
  - 주요 앱 (유튜브, 크롬, 사파리, 트위터) 테스트
  - 앱 내 수동 추가 기능도 제공
