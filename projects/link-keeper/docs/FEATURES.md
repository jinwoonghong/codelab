# Link Keeper 주요 기능 리스트

## 📋 기능 개요

Link Keeper는 4개의 개발 단계(Phase)로 나누어 점진적으로 기능을 구축합니다.

---

## ✅ Phase 1: MVP (Minimum Viable Product)

**목표**: 기본적인 링크 관리 기능 구현
**예상 기간**: 1주

### 1.1 링크 추가
- [ ] URL 입력 필드
- [ ] 자동 메타데이터 추출
  - 페이지 제목 (og:title 또는 title 태그)
  - 썸네일 이미지 (og:image)
  - 파비콘
  - 도메인 이름
- [ ] 수동 제목 입력 (메타데이터 추출 실패 시)
- [ ] 메모/설명 입력 (선택 사항)
- [ ] 저장 버튼 및 취소 버튼
- [ ] 입력 검증 (유효한 URL인지 확인)
- [ ] 중복 URL 방지

**사용자 시나리오**:
```
1. 사용자가 + 버튼 클릭
2. 링크 추가 모달이 나타남
3. URL을 입력하면 자동으로 제목과 썸네일 가져오기
4. 필요시 메모 추가
5. 저장 버튼 클릭
6. 목록에 새 링크가 추가됨 (토스트 메시지 표시)
```

### 1.2 링크 목록 보기
- [ ] 날짜별 자동 그룹화
  - 오늘
  - 어제
  - 이번 주
  - 이번 달
  - 이전
- [ ] 링크 카드 디자인
  - 썸네일 이미지
  - 제목
  - 도메인 이름
  - 저장 시간 (상대 시간: 5분 전, 2시간 전)
  - 읽음/안 읽음 상태 표시 (체크박스 또는 색상)
- [ ] 무한 스크롤 또는 페이지네이션
- [ ] 빈 상태 UI (링크가 없을 때)

**화면 예시**:
```
┌─────────────────────────┐
│    Link Keeper  🔍      │
├─────────────────────────┤
│ [전체] [안읽음] [읽음]   │
├─────────────────────────┤
│ 오늘                     │
│ □ [📷] 10 Amazing Tip... │
│       youtube.com • 5분전│
│ ✓ [📷] How to Code...    │
│       blog.com • 2시간전 │
├─────────────────────────┤
│ 어제                     │
│ □ [📷] Breaking News...  │
│       news.com • 어제    │
└─────────────────────────┘
       [+] 추가 버튼
```

### 1.3 링크 상태 관리
- [ ] 읽음 처리 (체크박스 또는 버튼)
- [ ] 안 읽음으로 되돌리기
- [ ] 읽음 상태에 따른 시각적 구분
  - 읽은 링크: 흐린 색상 또는 체크 아이콘
  - 안 읽은 링크: 선명한 색상 또는 동그라미 아이콘
- [ ] 읽은 시간 기록

### 1.4 링크 삭제
- [ ] 개별 링크 삭제
- [ ] 삭제 확인 다이얼로그
- [ ] 스와이프 제스처로 삭제 (선택)
- [ ] 삭제 취소 (Undo) 기능 (선택)

### 1.5 필터링
- [ ] 전체 보기
- [ ] 안 읽은 것만 보기
- [ ] 읽은 것만 보기
- [ ] 활성 필터 표시 (탭 또는 버튼)

### 1.6 로컬 데이터 저장
- [ ] IndexedDB 연동
- [ ] CRUD 연산 구현
  - Create: 링크 추가
  - Read: 링크 목록 조회
  - Update: 읽음 상태 변경
  - Delete: 링크 삭제
- [ ] 데이터 무결성 보장
